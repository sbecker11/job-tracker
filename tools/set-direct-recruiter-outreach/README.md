# SetDirectRecruiterOutreach — inline tri-state edits from a static HTML page

Browsers cannot write to sqlite from a `file://` page. This tiny Mac app
registers the custom URL scheme:

```
setdro://set?key=<normalized_key>&value=<yes|no|undecided>
```

and shells out to the `set-direct-recruiter-outreach` console script to
persist the change to `leads.db`.

Used by the `direct_recruiter_outreach` `<select>` in every funnel table on
`var/pending-actions.html` (generated by `scripts/render_pending_actions.py`
— see `directRecruiterCellHtml()`/`setDirectRecruiterOutreach()`): picking a
new option fires this scheme immediately, no separate "save" step.

Fire-and-forget, same as `refreshpending://` — a static page can't get a
return value back from the helper, so the page applies the new selection's
CSS class optimistically and trusts this app's own native alert to surface
a failure (bad key, DB locked, etc.).

## Install (once)

```bash
cd job-tracker
pip install -e .   # registers the set-direct-recruiter-outreach console script
cd tools/set-direct-recruiter-outreach
./install.sh
```

Installs `~/Applications/SetDirectRecruiterOutreach.app` and registers the
scheme with Launch Services. No Dock icon (LSUIElement).

## Smoke test

```bash
open 'setdro://set?key=acme::software engineer&value=yes'
python3 -c "
import sqlite3
conn = sqlite3.connect('var/leads.db')
print(conn.execute(\"SELECT direct_recruiter_outreach FROM job_leads WHERE normalized_key='acme::software engineer'\").fetchone())
"
```

Or just pick a different option from any star/dash dropdown in
`pending-actions.html` after re-rendering:

```bash
python scripts/render_pending_actions.py --no-rescore
```

The first browser click may show "Open SetDirectRecruiterOutreach?" —
allow it.

## Rebuild after edits

```bash
./install.sh
```
