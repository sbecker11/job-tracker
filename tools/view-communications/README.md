# ViewCommunications — view a lead's full message history from the dashboard

Browsers cannot query sqlite or open a PDF from a `file://` page. This tiny
Mac app registers the custom URL scheme:

```
viewcomms://open?company=<url-encoded>&title=<url-encoded>
```

and shells out to the `export-communications` console script
(`export-communications --db <leads.db> --company <company> --title
<title>`), which renders that lead's full `job_conversations` history
(every archived inbound/outbound message) to a fresh PDF under that lead's
package folder, then opens the PDF in Preview.

Used by the 💬 badge next to the message count in every funnel table on
`var/pending-actions.html` (generated by `scripts/render_pending_actions.py`
— see `titleCellHtml()`/`commsUrl()`): the badge only appears when a lead
has at least one archived conversation, and clicking it always regenerates
the PDF fresh from whatever's in `leads.db` right now (same as running
`export-communications` by hand — see `job-tracker/README.md`'s
"Review stored leads" section).

If a lead somehow has zero conversations by the time you click (rare —
`export-communications` would say "nothing to export"), or the export CLI
fails outright, this shows a native alert with whatever the CLI reported,
since a static page can't get a return value back from a fire-and-forget
custom-scheme call.

## Install (once)

```bash
cd job-tracker
pip install -e .   # registers the export-communications console script
cd tools/view-communications
./install.sh
```

Installs `~/Applications/ViewCommunications.app` and registers the scheme
with Launch Services. No Dock icon (LSUIElement).

## Smoke test

```bash
open 'viewcomms://open?company=Acme&title=Software%20Engineer'
```

Or just click a 💬 badge in `pending-actions.html` after re-rendering:

```bash
python scripts/render_pending_actions.py --no-rescore
```

The first browser click may show "Open ViewCommunications?" — allow it.
